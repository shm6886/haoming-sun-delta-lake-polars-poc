# 真实世界应用案例

## 案例 1：金融交易处理

**场景：** 银行处理每天百万笔交易

```
原始交易日志 (Bronze)
    ↓
去重、验证、掩码卡号 (Silver)
    ↓
按客户合并、计算统计 (Gold)
    ↓
风险评分、报表、合规检查
```

**关键：** 用 merge 处理实时更新，用时间旅行做审计

## 案例 2：用户行为分析

**场景：** 电商追踪用户浏览和购买行为

```
事件日志 (Bronze)
    - 页面浏览
    - 搜索查询
    - 购买记录
    ↓
去重、去异常、特征工程 (Silver)
    ↓
用户分群、推荐系统
```

**关键：** append-only Bronze 保留完整历史，用于重放/调试

## 案例 3：模型训练数据管道

**场景：** 准备 ML 模型的训练数据

```
多源数据 (日志、API、DB)
    ↓
标准化、缺失值处理 (Bronze)
    ↓
特征工程、样本均衡 (Silver)
    ↓
模型训练
```

**关键：** 版本控制确保可重现实验，时间旅行追踪数据来源

## 案例 4：数据质量监控

**场景：** 监控进入生产的数据质量

```
原始数据 (Bronze)
    ↓
质量检查
    - 缺失值
    - 异常值
    - Schema 变化
    ↓
质量报告 + 告警
```

**关键：** 用 merge 更新质量指标，用历史版本比较趋势

## 案例 5：跨域数据集成

**场景：** 合并来自多个系统的数据

```
系统 A (客户信息)
系统 B (订单信息)
系统 C (库存信息)
    ↓
标准化和清洁 (Bronze)
    ↓
主数据管理 (Silver)
    ↓
单一客户视图
```

**关键：** merge 处理重复客户，时间旅行追踪变化

## 生产部署检查清单

- [ ] 配置好 AWS 凭证和 IAM 权限
- [ ] 数据按日期分区以加快查询
- [ ] 设置 Bronze 长期保留（1 年+）
- [ ] 设置 Silver 合理保留（按业务需求）
- [ ] 实现监控和告警
- [ ] 文档化 ETL 逻辑
- [ ] 设置版本回退流程
- [ ] 定期测试灾难恢复

- [ ] 建立数据字典和血缘追踪
